gitの話

VScode上での作業を想定しています．
設定など終わったら，今回用のフォルダを作成します．（エクスプローラー上で適当に作ってください）
そのフォルダをVScode上で開きます．（面倒なので，そのフォルダしか開いていない状況が望ましいです）
その上で，
1.git clone [先ほどのURL]   （[]はいらない）
2. cd sakura_d_sashiniko　（作業フォルダへの移動）
3. git checkout -b ”ブランチ名”　（""はいらないです．名前は自由．自分の名前などが被らなくていいかも．）
4. ここで編集作業　ファイルの編集や追加など
5. git add -A
6. git commit -m "コメント"　（""が要ります．中身はなにか編集内容のコメント）（ブランチを作った最初の一回目のみ，これの代わりに，　git push --set-upstream origin ブランチ名　としてください）
編集をまたしたくなったら，編集した後，5,6を行う
7. git push
8. githubに移動（ブラウザ）
9. Compare & pull requestをクリック（これが出てこない場合もあります．7がうまくいったのに出てきていなければ，上部のPull requestsタブに移動して，下の方にある自分のブランチから，pull requestをしてください）
10. マージ（変更内容をくっつける作業）が出来たらgithubのmainへの保存完了

ここまでで一連の作業は終わりですが，自分が行った変更以外にも，他の人が行った変更を持ってくるために，続いて
11. git fetch origin
12. git rebase origin/main
これでローカル側にも行われた変更内容を持ってくることができます．（つまり現状の最新版を持ってこれる）
ここから，編集作業して，また5,6…みたいな感じです

変更内容にコンフリクト（変更内容が被っていたりして矛盾が生じる場合など）が生じる場合，githubがpuchとマージ段階でそれをお知らせしてくれます
それがあったらすぐに共有したほうがいいです

という感じで現状自分はうまくいっていますが，抜けている情報，間違っている部分などあれば教えていただけると嬉しいです．
